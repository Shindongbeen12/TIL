## **2주차(3.15)**
- ### **클라우드 개념 개요(1)**
  **1.IT 서비스란?**
  ```
  1.컴퓨터 기술을 이용해 기업이나 개인에게 제공하는 서비스
  2.인터넷과 스마트폰의 발전으로 대부분의 IT 서비스는 웹(web)이나 앱(App)을 통해 제공
  ```

    **2.웹 서비스를 만들어보자_IT서비스**

  - **수업이나 과제로 웹 서비스를 만드는 경우**
  - **상용 웹 서비스 환경**
  
    ```
    ★★
    - 컴퓨터 장치 사이에 서로 데이터를 주고받는 통신을 하려면 서로 간에 연결이 필요함
    - 연결 -> 네트워크 
    ```

  - **internet을 사용해 무선이든 유선이든 미국에서 서울까지 웹 서비스를 제공하는 것이 가능하다**


   **3.IT서비스와 인터넷**
    
    - **네트워크(Network)는 컴퓨터 장치들 사이를 연결하는 기술을 의미**
    - **인터넷(Internet)은 전세계의 작은 네트워크들이 모여 하나의 큰 네트워크를 구성한 것!**
    
    
    **4.IT서비스 구성**
    
    - **IT 서비스를 만들기 위해 필요한 것**
      
      **1.Internet(Network)**
      
      **2.Client(SW+HW)**
        
        - **예시)Web Browser + Computing Device**
      
      **3.Server(SW+HW)**
        - **예시)Web Application + Server**

    **5.IT 서비스 운영**
        
    **1. 상용 IT 서비스에 요구되는 점**
         
    **- 24시간/7일 내내 안정적인 서비스 제공**
    
    
    **2. 오랜 시간 안정적으로 동작하기 위해서 필요한 것**
        
    **1).소프트웨어 시스템 중단으로 이어지지 않아야 함**
          
    - **버그 없는 소프트웨어가 필요 -> 현실에 없음**
    - **소프트웨어공학적 접근을 통해 버그를 줄이고,버그기 실패로 이어지지 않는 아키텍처를 구상**
      
    **2).비정상 종료되지 않은 인프라(infrastructure)**
        
    - **인프라란 서버와 서버 간 연결을 위한 네트워크 등을 위한 SW 및 HW 장비를 가리킴**
    - **A) 장비 자체의 높은 안정성 필요**
    - **B) 안정적인 전원 공급**
      
      - **정전 등에 대비가 필요**
    - **C)온도 및 습도 관리**
      
      - **서버 사용량이 높아질수록 온도 상승**
      - **온도가 일정 수준 이상 높아지면 비정상 종료 가능**
      - **이 상태로 24시간 7일 내내 운영해야 함**
      - **습도에 민감하여,낮은 습도를 유지해야 함**
      
      **데이터 센터(Data Center)의 필요성** 
    
    **3).데이터 센터(Data Center)**
    
    - **인프라를 안정적으로 관리하기 위한 시설**
    - **정전이 발생하더라도 자체 전력설비를 통해 일정 시간 전원 공급 가능**
    - **하드웨어 인프라에 대한 발열 및 습도 관리**
        ```
        서버 랙(Server Rack)
        타워형 서버
        랙마운트형 서버
            1.밀집도가 더 높음
            2.같은 공간에 더 많은 서버 설치 가능
            3.더 많은 열 발생과 높은 전기 사용량
            4.온도/습도 조절 필요 전력 공급 및 관리 필요
        ``` 
    - **하드웨어 인프라에 대한 발열 및 습도 관리**
    - **네트워크를 통한 연결**
      ```
      - 네트워크란 컴퓨터와 컴퓨터를 연결하는 것을 의미
        - 컴퓨터 간에 물리적으로 케이블을 통해 연결
      - 1) 데이터 센터 내에 위치한 서버들을 서로 연결
      - 2) 데이터 센터 외부와 연결
      ``` 
    - **데이터 센터 VR투어**
    
    - **IT 서비스 단원 요약**
        ```
        서비스용 SW ---> [서버 인프라,네트워크] ---> 데이터센터
        ``` 
- ### **클라우드 개념 개요(2)**
  - **학습목표**
    
    **1).상용 IT 서비스를 구축하는 과정을 통해 클라우드 서비스의 이점을 알게 된다.**
    
    **2).클라우드 서비스가 급격히 확산된 이유를 알게 된다.**

    **1. 전통적인 IT 서비스 구축 과정**
        
        
        새로운 서비스를 해보자! -> 서버 등의 장비 구매
        ->데이터 센터 공간 임대 -> 데이터 센터 내 서버에 인터넷을 통해 접속 후 개발 진행 -> 서비스 시작
        
  
  **2. 전통적인 IT 서비스 구축 과정의 어려움**
      
    - **IT 서비스 구축 기간을 지연시키는 이슈**
         
         **1). 서버 등의 장비 구매 과정: 수개월 이상 걸리는 경우가 많음**
         
         **2).데이터 센터 직접 구축하는 경우**

    - **큰 비용이 발생하는 경우**
      
        **1).서버 등의 장비 구매**
        ```
        - 장비를 어떤 스펙으로 구매해야 할까?
          얼마나 구매해야 할까?
        - 사용자 -> 서버 인프라
           사용자 규모 예측에 성공할 경우 
           안정적 서비스 제공 및 비용 최적화
        ``` 
        **2).데이터 센터 직접 구축하는 경우 역시 큰 비용 발생**
    
    **3.클라우드 환경에서 IT 서비스 구축 과정**
        
    - **장비 구매 및 데이터 센터 준비 과정이 없어짐**
        ```
        새로운 서비스를 해보자! -> 서버 등의 장비 대여 요청
        -> 클라우드 컴퓨팅 서비스 -> 개발 진행 
                        -> 서비스 시작
        ```  
        ```
        1.데이터 센터 내 미리 준비한 IT 인프라
            *사용자가 대여하게 됨
        2.클라우드 서비스 제공자가 보유한 데이터 센터
        3.클라우드 컴퓨팅 서비스
        ```
    - **클라우드 환경에서 IT 서비스를 구축할 때의 장점**
    
    **1). 낮은 초기 비용과 구매 기간 단축**
        
    - **서버나 장비 구매가 불필요(가상화 기술로 인한 이유)**
    - **클라우드 서비스 제공자로부터 필요한 서버나 장비를 가상화 형태로 대여**
        
    - **원하는 때에 원하는 만큼 장비 대여 가능 
         -> [On-Demand]**
    - **사용한 만큼 비용을 지불 -> 종량제 결제 모델(pay-as-you-go)** 

   **2).사용자 규모 예측 불필요**
   - **사용자 규모가 증가하는 경우 즉시,필요한 만큼의 자원을 추가로 대여 -> 확장성**
   - **사용자 규모가 줄어드는 경우 반납 가능 -> 탄력성 -> 비용 최적화 가능** 
    
    **4. 서버 가상화(Server Virtuakization)**
   
   - **서버 가상화**
     
     - **사용자가 서버를 대여하려 할 때,사람이 직접 서버를 할당해주는 것이 아님**
     - **서버 가상화 기술을 통해 물리 서버에서 가상머신을 실행하여 사용자에게 제공**
     - **사용자는 물리 서버를 사용하는 것과 차이를 알 수 없음**
     ```
     사용자 -> 물리 서버(Physical Server)
     사용자 -> 가상 머신(Virtual Machine)
     ```   
   - **사용자가 원하는 때에 원하는 만큼의 서버를 대여할 수 있는 것은 가상화 기술이 있기 때문**

    **5. 클라우드 환경의 확장성**
   
   - **서버 및 장비를 원하는 만큼 사용하는 것이 가능 -> On-Demand**
   - **필요한 경우 추가 대여하는 것이 가능 -> 확장성(Scalability)**
        ```
        -사용자가 늘어날수록 서버에 과부하
            -> 서비스 장애 발생
        -필요로 하는 만큼 원하는 때에 
         사용하는 서버를 늘릴 수 있음
        ```
        ```
        - 사용자가 다시 줄어드는 경우 대여한 자원을 반납
        
        -> 탄력성(Elasticity)
        -> 비용 최적화(Cost Optimization)
        
        -사용자가 늘어날수록 서버에 과부하
            -> 서비스 장애 발생
        -필요로 하는 만큼 원하는 때에 
            사용하는 서버를 늘릴 수 있음
        ```    
    **6.클라우드 서비스 제공자**
    
  - **클라우드 컴퓨팅 서비스를 제공하는 업체를 CSP라고 함**
  - **대표적인 클라우드 서비스 제공자**
